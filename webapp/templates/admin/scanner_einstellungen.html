Alte Scan-Daten bereinigen

Bei täglichen Scan-Läufen wächst die idv_file_history-Tabelle stark an, da für jede Datei ein History-Eintrag geschrieben wird (auch bei unverändertem Inhalt). Die Bereinigung löscht:

  • Alle History-Einträge älter als N Tage
  • Scan-Läufe älter als N Tage, die nicht mehr als letzter Scan einer Datei referenziert werden
Scan-Läufe, die noch als last_scan_run_id einer aktiven Datei eingetragen sind, werden nicht gelöscht — auch wenn sie älter als N Tage sind.

Scanner-Datenbank importieren

Wurden Scans auf anderen Rechnern mit einer eigenen Datenbank durchgeführt (db_path in config.json auf eine lokale Datei gesetzt), können die Ergebnisse hier in die zentrale idvault-Datenbank zusammengeführt werden.
Vorhandene Pfade werden nicht dupliziert – neuere Scan-Daten überschreiben ältere.

Typischer Ablauf:
  1. Scanner auf Rechner B mit eigenem db_path laufen lassen, z. B. scan_B.db
  2. scan_B.db auf den Server kopieren (z. B. per USB / Netzwerk)
