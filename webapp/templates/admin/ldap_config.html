Server-Konfiguration
Wenn aktiviert, werden alle Benutzer zuerst gegen LDAP geprüft. Lokale Logins ohne Passwort-Hash bleiben weiterhin möglich (Notfall-Admin).
Protokoll ldaps:// für verschlüsselte Verbindung (empfohlen)
Standard: 636 (LDAPS) oder 389 (LDAP)
Suchbasis für Benutzer im Verzeichnis
Service-Account für LDAP-Suche (nur Leserechte benötigt)
Wird verschlüsselt gespeichert (Fernet/AES-128). {% if cfg and cfg.bind_password %} Kennwort ist hinterlegt. {% endif %}
sAMAccountName = Anmeldename (empfohlen für AD)
Deaktivieren nur bei selbstsignierten Zertifikaten im Test
Login-Ablauf
  1. Benutzer gibt Anmeldename + Passwort in idvault ein
  2. idvault verbindet via LDAPS mit dem Server
  3. Suche nach Benutzerkonto per sAMAccountName
  4. LDAP-Bind mit User-DN + eingegebenem Passwort
  5. Gruppen-Mitgliedschaft → idvault-Rolle
  6. Person wird automatisch angelegt/aktualisiert
